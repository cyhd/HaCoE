22 /**********************************************************************************************************
23 HapticThreadHaptReplication class
24 
25 Created: May 2013
26 Author: Devin Luu, student ict, CSIRO
27 
28 This class is used to control perform a passive haptic replication experiment.
29 
30 HaptDeviceA will produce a force which the subject must resist.
31 HaptDeviceB will hold its position and the subject must replicate the resisting force on HaptDeviceB.
32 
33 After the user chooses which haptic devices to activate and presses start on the GUI, and if the user
34 chooses to run the Haptic Replication experiment, this thread will run.
35 
36 Both Entact devices must be activated for this thread to run properly.
37 
38 This thread does not instantiate the haptic devices. It only manages the control aspects (gathering sensor
39 information, calculation of force vectors, and commanding motion of the haptic devices). The main
40 functionality of the thread is implemented in the run function.
41 
42 The thread will poll the GUI through haptlinksupervisor for the stop command while running. If a stop command
43 is detected, then the thread stops.
44 *************************************************************************************************************/
